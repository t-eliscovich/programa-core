Anular cheque por error de carga

Marca el cheque como mal cargado y aplica las compensaciones automáticamente. Distinto de "marcar rebotado": no marca el cliente en STOP. Después podés crear el cheque correcto.

Side-effects que se aplicarán:
  • El cheque pasa a stat='X' con tag [X] error de carga en observación.
    • {% set stat = (ch.stat or '')|upper %} {% if stat in ['B', 'V', 'W', 'I', 'J', 'K', 'A'] %}
  • Se inserta una compensación ND en el banco con saldo running recalculado.
    • {% elif stat == 'C' %}
  • Se inserta una salida en caja con saldo running recalculado.
    • {% else %}
  • No tenía movimiento en banco/caja — sólo se actualiza el cheque.
    • {% endif %}
  • Si tiene aplicaciones a facturas, se reversa el abono/saldo.
  • Si tiene posdat hermana, se elimina.
  • NO marca al cliente en STOP (es error administrativo).
